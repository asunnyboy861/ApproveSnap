## 2. 痛点研究与客户需求

### 2.1 核心痛点总结

基于Reddit、Hacker News、Twitter、行业博客的多语言深度调研，项目审批工作流领域存在以下6大核心痛点：

#### 痛点1: 审批散落在邮件链中 (Pain Score: 95/100)

> "People are paying $50/month for project management, then using email chains for approvals."  
> — Reddit r/SaaS

> "Approvals are scattered across emails, Slack, and WhatsApp. Your clients will eventually lose your approval request in their mailboxes."  
> — SocialPilot Research

**具体表现**:
- 审批请求通过邮件发送，客户回复散落在不同邮件线程中
- 附件版本混乱，没人知道哪个是最终版
- "Looks good" 这种模糊回复无法作为正式审批记录
- 邮件搜索困难，历史审批难以追溯

#### 痛点2: 客户需要登录不熟悉的工具 (Pain Score: 92/100)

> "The client has to log in to a tool they don't use. Most of the time, approvals happen on texts and emails."  
> — SocialPilot Research

> "ApproveThis was built for the approvals that cross organizational lines — the ones where the person who needs to say 'yes' doesn't work for you, doesn't want another login."  
> — ApproveThis

**具体表现**:
- 客户不愿意为了一次审批创建账号
- 登录流程成为审批的摩擦点
- 客户忘记密码或找不到登录入口
- 每增加一个步骤，审批响应率下降30%+

#### 痛点3: 没有审批截止日期，无限等待 (Pain Score: 90/100)

> "If there's no set approval window, the client ends up controlling your schedule. A request without a deadline is just another task on their list."  
> — SocialPilot Research

> "Projects don't miss deadlines because of execution. They miss because clients take two weeks to review something that should take two days."  
> — AgencyPro

**具体表现**:
- 审批请求没有截止时间，客户无限拖延
- 没有自动提醒机制
- 团队等待审批期间无法推进工作
- 3-6周的"死时间"成为项目延期主因

#### 痛点4: 没有审计追踪，纠纷无据可查 (Pain Score: 88/100)

> "Email isn't a formal record. If a client dispute arises post-delivery, you're left reconstructing conversations."  
> — Noloco

> "Nothing to show when a client claims they never signed off on a caption weeks later."  
> — SocialPilot Research

**具体表现**:
- 审批记录不完整，无法证明客户已同意
- "I never approved that" 纠纷频发
- 范围蔓延（Scope Creep）因为审批不明确
- 财务纠纷——客户拒绝付款声称未审批

#### 痛点5: 反馈分散在多个渠道 (Pain Score: 85/100)

> "Client feedback often arrives across multiple reply threads, WhatsApp messages, and meeting notes. Someone has to manually consolidate this."  
> — Noloco

> "The client reviews a post in 90 seconds. The approval cycle around that 90 seconds costs your team an average of 35 to 45 minutes per post."  
> — ZoomSphere

**具体表现**:
- 一部分反馈在邮件，一部分在Slack，一部分在电话
- 团队需要手动整合来自多渠道的反馈
- 反馈与具体版本不对应
- 35-45分钟/条的审批管理成本

#### 痛点6: 现有工具功能过剩且价格高 (Pain Score: 82/100)

> "The word 'overkill' shows up in more complaint threads than any other word."  
> — Reddit r/microsaas

> "Sales proposals need three to five days to clear approvals. Half the team's time goes to managing the approval process itself."  
> — Operations Upgrade

**具体表现**:
- Monday.com/Asana/ClickUp等工具审批功能弱，需要额外工具
- 专门的审批工具（如ApproveThis）$50+/月，对自由职业者太贵
- 大多数审批工具只有Web版，没有原生iOS应用
- 功能过剩：小团队不需要复杂的多级审批、条件分支等

### 5.3 魔法链接审批系统（核心差异化技术）

```swift
import Foundation
import CryptoKit

final class MagicLinkService {
    
    private let baseURL = "https://approvesnap.app/approve"
    private let secretKey = SymmetricKey(size: .bits256)
    
    func generateMagicLink(for project: Project, expirationHours: Int = 72) -> String {
        let payload = MagicLinkPayload(
            projectId: project.id.uuidString,
            clientEmail: project.clientEmail,
            expiresAt: Date().addingTimeInterval(TimeInterval(expirationHours * 3600))
        )
        
        guard let encoded = try? JSONEncoder().encode(payload) else { return "" }
        
        let sealedBox = try? AES.GCM.seal(encoded, using: secretKey)
        guard let token = sealedBox?.combined.base64EncodedString() else { return "" }
        
        let safeToken = token
            .replacingOccurrences(of: "+", with: "-")
            .replacingOccurrences(of: "/", with: "_")
            .replacingOccurrences(of: "=", with: "")
        
        return "\(baseURL)?token=\(safeToken)"
    }
    
    func validateMagicLink(token: String) -> MagicLinkPayload? {
        let paddedToken = token
            .replacingOccurrences(of: "-", with: "+")
            .replacingOccurrences(of: "_", with: "/")
        
        let padding = 4 - (paddedToken.count % 4)
        let fullToken = padding < 4 ? paddedToken + String(repeating: "=", count: padding) : paddedToken
        
        guard let data = Data(base64Encoded: fullToken),
              let sealedBox = try? AES.GCM.SealedBox(combined: data),
              let decrypted = try? AES.GCM.open(sealedBox, using: secretKey),
              let payload = try? JSONDecoder().decode(MagicLinkPayload.self, from: decrypted) else {
            return nil
        }
        
        guard payload.expiresAt > Date() else { return nil }
        
        return payload
    }
}

struct MagicLinkPayload: Codable {
    let projectId: String
    let clientEmail: String
    let expiresAt: Date
}
```

### 5.4 审批工作流状态机

```swift
import Foundation

final class ApprovalStateMachine {
    
    private let validTransitions: [ApprovalItem.ApprovalStatus: Set<ApprovalItem.ApprovalStatus>] = [
        .pending: [.approved, .rejected, .changesRequested, .expired],
        .changesRequested: [.pending, .approved, .rejected],
        .approved: [],
        .rejected: [.pending],
        .expired: [.pending]
    ]
    
    func transition(
        from current: ApprovalItem.ApprovalStatus,
        to next: ApprovalItem.ApprovalStatus
    ) throws -> ApprovalItem.ApprovalStatus {
        guard let allowed = validTransitions[current],
              allowed.contains(next) else {
            throw ApprovalError.invalidTransition(from: current, to: next)
        }
        return next
    }
    
    func canTransition(
        from current: ApprovalItem.ApprovalStatus,
        to next: ApprovalItem.ApprovalStatus
    ) -> Bool {
        validTransitions[current]?.contains(next) ?? false
    }
}

enum ApprovalError: LocalizedError {
    case invalidTransition(from: ApprovalItem.ApprovalStatus, to: ApprovalItem.ApprovalStatus)
    
    var errorDescription: String? {
        switch self {
        case .invalidTransition(let from, let to):
            return "Cannot transition from \(from.rawValue) to \(to.rawValue)"
        }
    }
}
```

### 5.5 通知与提醒引擎

```swift
import UserNotifications
import Foundation

final class NotificationService: NSObject, UNUserNotificationCenterDelegate {
    
    static let shared = NotificationService()
    
    func requestAuthorization() async throws -> Bool {
        try await UNUserNotificationCenter.current()
            .requestAuthorization(options: [.alert, .badge, .sound])
    }
    
    func scheduleApprovalReminder(
        for project: Project,
        deadline: Date,
        reminderIntervals: [TimeInterval] = [86400, 43200, 7200]
    ) {
        for (index, interval) in reminderIntervals.enumerated() {
            let triggerDate = deadline.addingTimeInterval(-interval)
            guard triggerDate > Date() else { continue }
            
            let content = UNMutableNotificationContent()
            content.title = "Approval Reminder"
            content.body = "\(project.clientName) hasn't approved \"\(project.name)\" yet. Deadline: \(deadline.formatted(date: .abbreviated, time: .shortened))"
            content.sound = .default
            content.categoryIdentifier = "APPROVAL_REMINDER"
            content.userInfo = ["projectId": project.id.uuidString]
            
            let components = Calendar.current.dateComponents(
                [.year, .month, .day, .hour, .minute],
                from: triggerDate
            )
            let trigger = UNCalendarNotificationTrigger(dateMatching: components, repeats: false)
            
            let request = UNNotificationRequest(
                identifier: "reminder-\(project.id.uuidString)-\(index)",
                content: content,
                trigger: trigger
            )
            
            UNUserNotificationCenter.current().add(request)
        }
    }
    
    func sendApprovalReceivedNotification(for project: Project, action: AuditLog.AuditAction) {
        let content = UNMutableNotificationContent()
        
        switch action {
        case .approved:
            content.title = "Approved!"
            content.body = "\(project.clientName) approved \"\(project.name)\""
        case .rejected:
            content.title = "Rejected"
            content.body = "\(project.clientName) rejected \"\(project.name)\""
        case .changesRequested:
            content.title = "Changes Requested"
            content.body = "\(project.clientName) requested changes on \"\(project.name)\""
        default:
            return
        }
        
        content.sound = .default
        content.categoryIdentifier = "APPROVAL_UPDATE"
        content.userInfo = ["projectId": project.id.uuidString]
        
        let request = UNNotificationRequest(
            identifier: "update-\(project.id.uuidString)-\(Date().timeIntervalSince1970)",
            content: content,
            trigger: nil
        )
        
        UNUserNotificationCenter.current().add(request)
    }
    
    func cancelReminders(for projectId: UUID) {
        let center = UNUserNotificationCenter.current()
        center.getPendingNotificationRequests { requests in
            let ids = requests
                .filter { $0.identifier.hasPrefix("reminder-\(projectId.uuidString)") }
                .map(\.identifier)
            center.removePendingNotificationRequests(withIdentifiers: ids)
        }
    }
}
```
